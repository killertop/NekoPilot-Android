# NekoPilot UI 完整性审计

审计日期：2026-07-15  
审计对象：当前 `codex/go-core-migration` 分支的 Debug/QA 构建  
审计方式：页面清单、布局资源扫描、AndroidManifest 入口核对、模拟器逐页实机检查

## 结论

方案 1 的 UI 补齐工作已完成。主框架、四个一级页面、共享参数编辑器、设置、分组、路由、日志、工具、备份、应用选择、规则资产、扫码导入和关于页均已接入统一的 NekoPilot 视觉体系，并完成 1080 × 2335 中文浅色主题实机回归。

本轮同时修复了全局下拉偏好控件缺少 Spinner 导致的崩溃、二级页面底部导航误导、日志默认信息过载和关于页版本信息换行等问题。当前未发现 P0、P1 或 P2 遗留项。

## 页面状态

### 已完成

- 连接首页：状态、当前节点、实时流量、启动按钮和底部导航已统一。
- 节点主列表：列表卡片、空状态、主要操作和底部导航已统一。
- 规则主列表：规则卡片、说明和主要操作已统一。
- 一级导航框架：连接、节点、规则、设置四个入口已建立。
- 全局工具栏、颜色、圆角、明暗主题基础令牌已建立。

### 本轮补齐

- 设置与所有 Preference 共享编辑器统一为卡片、图标容器、章节标题和一致的开关/下拉控件。
- Shadowsocks、SOCKS、HTTP、VMess、VLESS、Trojan、Trojan-Go、Mieru、Naive、Hysteria、SSH、WireGuard、TUIC、ShadowTLS、AnyTLS、链式代理和自定义配置等共享参数页全部随全局样式迁移。
- 分组列表、分组编辑器、路由编辑器、规则资产和分应用代理完成统一。
- 日志增加全部、警告、错误筛选及可读空状态，默认突出警告。
- 网络工具、备份恢复、扫码、导入、原始配置编辑和关于页完成重构。
- 侧边抽屉收敛为分组、日志、工具、关于，移除与底部导航重复及推广入口。
- 二级页面自动隐藏底部导航，避免错误表达当前一级标签状态。

## 遗留优先级

- P1：无。

- P3：极长的 sing-box 功能版本字符串仍会自然换行，但不影响阅读或操作。

## 可访问性复核

- 关键图标按钮保留内容描述，主要触控目标不小于 48dp。
- 日志使用等宽字体、明确筛选和高对比度空状态。
- 长设置与编辑表单通过章节标题、卡片边界和一致的摘要层级帮助定位。
- 本轮覆盖标准字体和竖屏；超大动态字体、TalkBack 焦点顺序及横屏仍属于后续专项兼容性测试范围，不构成当前视觉补齐阻塞项。
