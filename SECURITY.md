# 安全策略 / Security policy

请不要公开提交或报告订阅链接、节点凭据、私钥、签名文件、签名口令或包含敏感信息的日志。
Do not publicly submit or report subscription links, node credentials, private keys, signing files, signing passwords, or logs containing sensitive information.

发现安全问题时，请通过 GitHub 账户的私密联系方式联系维护者，并提供最小化、已脱敏的复现材料。不要先创建公开 issue。
For security issues, contact the maintainer through a private GitHub-account channel with minimal, sanitized reproduction material. Do not open a public issue first.
